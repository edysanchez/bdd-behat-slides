BDD and Behat

BDD

What is BDD?

  • TDD extension
  • Write human-readable stories that describes the application behaviour
  • Focuses the communication between business developers and QA
  • Code is self documenting

Gherkin

Gherkin is the language used to define behaviour

Gherkin Depth levels

  • Feature
  • Scenario
  • Step

Feature

  • What the feature offers
  • Who will get the benefit
  • Why we want the feature
                            Feature: list folder content
                            In order to see the directory structure
                            As a OS user
                            I am able to list the current directory's contents

Scenario

Use case explanation

                            Scenario: list 1 file in the directory

Step

Scenario Step which can be for

  1. Set scenario context

    Given I am in the directory "test"

  2. Use the system

    When I run "ls"

  3. Check the system state

    Then I should get """" . """"

Other constructions

  • Background
  • Scenario Outline
  • TableNode

Behat

Behat is a PHP implementation of cucumber

links methods with gherkin through regexp in documentation comments

                        /**
                        *@When/^I run "ls"$/ 
                        */
                        public function iRun()

Behat

  • Tags for filtering at feature,scenario and step level
  • Hooks to execute code before or after Step/Scenario/Features
  • Different configuration profiles and suites configurables via tags

Behat

CheatSheet

  • -p profile executes the selected profile
  • --suite=suiteName execute the suite
  • file.feature[:lineNumber] Execute all scenarios in the feature file or the one starting in lineNumber
  • --dry-run simulates what will run
  • -dl shows which steps can recognize

Mink

Behat extension which offers a common interface to interact with different web drivers

Also offers a few predefined steps for basic assertions and browser operations
